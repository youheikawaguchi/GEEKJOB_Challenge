Java EE
	正式名称：Java Platform, Enterprise Edition
	※現在は、OracleからEclipse Foundationに移管されたため、「JakartaEE」に名称変更されている
	概要	：企業向けのWebアプリケーション開発用の機能をセットにしたもの
	
		類似⇒Java SE
		正式名称：Java Platform, Standard Edition
		概要	：標準的なJavaアプリケーションの開発に必要な機能だけをセットにしたもの
	
	JavaEEはJavaSEにWebアプリケーションに必要なサーバー関係のライブラリなどを追加したもの
	なので、JavaEEにはJavaSEが含まれている。
	
	JavaEEには二つのSDKがあり、
	1.フルプラットフォームプロファイル(通常版)
	2.Web Profile
	がある。
	
	特徴	：比較的実行速度が速く、安全性も高いので、多くのWebアプリケーションで使われている。
	SNS、ECサイト、銀行、予約サイトなどが挙げられ、TwitterやEvernoteもJavaで作られている。
	
Spring
	概要：従来のWebアプリケーションにおける問題点を解決しており、非常に有力なフレームワーク
	
	特徴：変更に強い、テストや容易、拡張性が高い、保守性が高い、再利用性が高い
	オブジェクト指向の基本原則を徹底して、Javaを最大限に生かすことが出来るようにされています。
	
	得意な分野：
	Webシステム
		柔軟性や拡張性があるため、大規模な開発や仕様変更に対応しやすい。
	Webサービス
		規模の拡大にも対応できるほどの、昨日がまんべんなく提供されている
		
Apache Struts
	特徴：整備されたJSPカスタムタグによってJavaコードはJSPファイルと分離され、
	従来のJSPのようにHTMLタグの中に<%と%>で囲まれたスクリプトレットであるJavaソースコードを
	混在させる必要なく読みやすく洗練されたコーディングができるようになっていた
	
	サーブレットから設定ファイル（struts-config.xml）に指定されたアクションクラスを呼び出し、アクションクラスの実行結果により、画面表示を決める事ができます。
	
	言語毎に 各画面項目、エラーメッセージなのを定義したapplication.properties （日本語用「application.properties」、英語用「application_en.properties」など）を用意し 設定ファイル（struts-config.xml）で切り替えることで、 言語毎にJSPを用意するのではなく、１つのJSPで多言語化することができる。 ファイル名の次に、“＿”と言語名を記載します。
	
	入力チェックを定義したフォ−ム定義ファイルの作成(validation.xml)を作成し 設定ファイル（struts-config.xml）でアクションフォームを指定し アクションフォームで入力値や結果値のデータの整合性等を確認する事ができます。
